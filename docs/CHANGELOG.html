BUG FIX Rate Creation/Update Authentication Issue

Date: November 2025
Status: ‚úÖ Fixed

Problem

Rates could not be created or updated through admin interface. Operations returned 401 Unauthorized errors.

Root Cause

Rate endpoints used auth.admin_required decorator with strict issuer validation. Token rejected when:

  • Frontend accesses Keycloak via http://192.168.1.222:8080
  • Backend expects issuer: http://keycloak:8080 (internal Docker name)

Solution

  1. Created new admin_required in secure_auth.py with flexible issuer validation
  2. Accepts multiple issuers:
    • http://keycloak:8080/realms/peppertree
    • http://localhost:8080/realms/peppertree
    • http://192.168.1.222:8080/realms/peppertree
    • http://127.0.0.1:8080/realms/peppertree
  3. Updated rate_routes.py to use new decorator

Files Modified

  • backend/secure_auth.py - Added flexible admin_required()
  • backend/rate_routes.py - Use secure_auth.admin_required
  • src/services/secureApiService.js - Added skipSecurePrefix option

Benefits

  • ‚úÖ Works from browser (any IP)
  • ‚úÖ Works from localhost
  • ‚úÖ Works from internal Docker network
  • ‚úÖ Maintains security (signature still verified)
  • ‚úÖ Consistent with other secure endpoints
